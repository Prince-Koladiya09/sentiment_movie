# 🧠 Sentiment AI — ML Backend

A production-quality machine learning backend for sentiment analysis on IMDB movie reviews.
This project focuses on a robust ML pipeline that trains multiple models, generates detailed evaluation metrics, and serves them via a FastAPI backend.

## 🏗️ Architecture

```
sentiment-movie/
├── train_and_export.py        ← ML Pipeline: Trains models & exports JSON data
├── debug_bert.py              ← Environment diagnostics (TensorFlow/Transformers)
├── backend/
│   ├── main.py                ← FastAPI Server: Serves model metrics & analysis
│   ├── requirements.txt       ← Backend & ML dependencies
│   ├── .env                   ← Environment configuration
│   └── data/exports/          ← Generated by train_and_export.py
│       ├── metrics.json        ← Accuracy, F1, AUC-ROC, etc.
│       ├── confusion_matrices.json
│       ├── roc_curves.json
│       ├── pr_curves.json
│       ├── training_history.json
│       ├── error_samples.json  ← Real misclassifications for analysis
│       ├── lime_examples.json  ← LIME word-level explanations
│       ├── confidence_dist.json ← Confidence vs Correctness analysis
│       ├── feature_importance.json ← Top predictive words
│       ├── dataset_stats.json  ← Sample reviews & corpus stats
│       └── model_agreement.json ← Inter-model correlation matrix
└── venv/                      ← Python Virtual Environment
```

## 🚀 Quick Start

### 1. Setup Environment

```bash
# Create and activate virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r backend/requirements.txt
```

### 2. Train Models & Export Data

Run the training pipeline once to build the models and generate the analysis data. This process auto-downloads the IMDB dataset if it's not present.

```bash
python train_and_export.py
```

**The pipeline trains 4 distinct models:**
- **Naive Bayes**: TF-IDF bigrams + MultinomialNB (Fast baseline).
- **Logistic Regression**: TF-IDF bigrams with sublinear scaling (Strong classical performer).
- **Bidirectional LSTM**: Keras-based deep learning with learned embeddings.
- **DistilBERT**: HuggingFace Transformer model (State-of-the-art accuracy).

> [!NOTE]
> Deep learning models (LSTM/DistilBERT) require `tensorflow` and `transformers`. If these are not installed, the script will skip them and the API will serve results for the classical models.

### 3. Start the API Server

Before starting, you can configure the backend via `backend/.env`:
```env
ENVIRONMENT=development
PORT=8000
ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000
```

Then run the server:
```bash
cd backend
uvicorn main:app --reload --port 8000
```
The API serves structured data to be consumed by any modern frontend dashboard.

## 📡 API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/` | `GET` | System status, version, and list of available data exports |
| `/metrics` | `GET` | Overall performance metrics (Accuracy, F1, etc.) |
| `/dataset/stats` | `GET` | Corpus statistics, word counts, and sample reviews |
| `/errors` | `GET` | Filterable list of model misclassifications |
| `/lime-examples` | `GET` | LIME explanations showing word-level influence |
| `/feature-importance` | `GET` | Global top positive/negative words (NB/LR) |
| `/model-agreement` | `GET` | Correlation matrix showing how often models agree |
| `/training-history` | `GET` | Epoch-by-epoch loss/accuracy for DL models |
| `/confusion-matrices` | `GET` | Row/column matrix data for model evaluation |
| `/roc-curves` | `GET` | False Positive vs True Positive rate data points |
| `/pr-curves` | `GET` | Precision-Recall curve data points |

## 🛠️ Tech Stack

- **ML Core**: `scikit-learn`, `numpy`, `pandas`, `nltk`
- **NLP**: `TfidfVectorizer`, `WordNetLemmatizer`, `LIME` (Explainable AI)
- **Deep Learning**: `TensorFlow`, `Keras`, `HuggingFace Transformers` (DistilBERT)
- **Backend Framework**: `FastAPI` (with Pydantic validation)
- **Server**: `Uvicorn`

## 📝 IMDB Dataset
The script utilizes the [IMDB Dataset of 50,000 Movie Reviews](https://www.kaggle.com/datasets/lakshmi25npathi/imdb-dataset-of-50000-movie-reviews). It is perfectly balanced with 25,000 positive and 25,000 negative samples.

The project is configured to:
1. Look for `IMDB Dataset.csv` in the root directory.
2. Auto-download via `datasets` library if the CSV is missing.
